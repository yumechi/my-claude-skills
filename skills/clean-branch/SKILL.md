# Clean Branch

mainブランチに切り替え、リモートを更新、マージ済みのローカルブランチを削除します。

> このスキルは独立したサブエージェントとして実行されます。実行内容を確認したうえで
> 提案内容を返し、メインエージェントがユーザー承認後に実際のクリーンアップを行います。
> 会話履歴は参照できないため、独立した判断で進行してください。

## 実行手順

### 1. 現在の状態確認

```bash
git status --porcelain
```

未コミット変更がないか確認し、あれば警告する。

### 2. mainブランチへ切り替え

```bash
git switch main
```

現在のブランチをmainに切り替えます。失敗した場合は、警告を表示して中止します。

### 3. リモート情報を更新

```bash
git pull && git fetch -p
```

- `git pull` - リモートの最新を取得してmainを更新
- `git fetch -p` - リモートで削除されたブランチ参照をローカルから削除（prune）

### 4. マージ済みブランチを確認・削除

```bash
gh poi
```

GitHub CLI の拡張機能 `gh poi` でマージ済みのローカルブランチを削除します。
この際、削除対象のブランチ一覧が表示されます。

## チェック項目

### 未コミット変更の確認

```bash
git status --porcelain | wc -l
```

出力が 0 でない場合は以下を警告：
- 現在のブランチに未コミット変更がある
- コミットするか、変更を破棄してから実行すること

### ブランチ切り替え可否

以下で確認：

```bash
git switch main 2>&1 | grep -E 'error|fatal'
```

エラーがある場合は以下の理由を調査：
- 現在のブランチに未コミット変更がある
- mainブランチが存在しない

### gh poi のインストール確認

```bash
gh extension list | grep poi
```

出力がない場合は未インストール。以下で導入：

```bash
gh extension install github/gh-poi
```

1. mainブランチに切り替え

- 削除対象ブランチの一覧（事前確認）

### 警告・注意

- 危険な処理やユーザー確認が必要な項目を明記

## 適用手順

メインエージェントがユーザー承認後に以下を実行：

```bash
git switch main && git pull && git fetch -p && gh poi
```

## 注意事項

- 現在のブランチに未コミット変更がある場合、`git switch main` が失敗する
- `gh poi` は GitHub CLI の拡張機能。インストール済みの前提
- mainブランチが存在しないと実行できない
- 削除対象の確認なしに全削除するため、内容をしっかり確認してから実行
